# Security & Compliance Basics (Test-only)

## Data Classification
- Public: safe to share externally.
- Internal: for employees only.
- Confidential: customer data, financials, unreleased product plans.
- Restricted: credentials, security keys, sensitive PII.

## Handling Rules
- Store Internal/Confidential data only in approved systems.
- Do not upload Confidential/Restricted data to public tools.
- Use encryption for external transfers when approved.

## Passwords and MFA
- MFA is mandatory on all corporate accounts.
- Password manager is required for storing credentials.

## Incident Reporting
If you suspect a security incident:
1. Disconnect affected device from network (if safe).
2. Notify IT & Security via the incident hotline or ticket.
3. Preserve evidence (do not delete files).

## Phishing
Report suspicious emails using the “Report Phishing” button in email client.
